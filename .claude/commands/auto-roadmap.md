# 自动执行 ROADMAP 任务

持续执行 ROADMAP.md 中的未完成任务，一直到全部完成位置。

创建链式 PR。

## 工作流程

### 初始化
记录当前分支为基础分支。

### 循环执行（直到没有未完成任务）

1. **读取任务**：找到 ROADMAP.md 中第一个 `- [ ]` 任务，无则结束
2. **实现功能**：
   - 分析任务，修改代码（遵循 ERRATA.md 和 CLAUDE.md）
   - 添加测试，运行 `moon test` 确保通过
   - 更新 ROADMAP（改为 `- [x]`，更新"下一步"）
3. **创建 PR**：
   - 运行 `moon fmt && moon info`
   - 基于当前分支创建新功能分支
   - 提交更改，推送并创建 PR（base 为前一个分支，首个为 main）
4. **继续**：保持在当前分支，回到步骤 1

## 分支策略

```
main
  └── feat/task-1 (PR #1 -> main)
        └── feat/task-2 (PR #2 -> task-1)
              └── feat/task-3 (PR #3 -> task-2)
```

每个 PR 只显示增量更改，按顺序合并即可。

## 输出

每个任务完成后输出 PR 链接。全部完成后输出 PR 列表（按合并顺序）。

## 注意

- 任务依赖未实现功能时跳过并说明
- 完成一个任务后开启下一个任务，不要停下来等待用户回复
- 每个任务完成后立即更新 ROADMAP.md
- 保持代码质量，不要为了完成任务而降低标准
- 参考 ERRATA.md 中的最佳实践
