# 处理勘误文件待办事项

请执行以下步骤：

## 1. 读取勘误文件

读取 `ERRATA.md` 文件，找到 `## 待补充` 部分的所有待办事项（以 `- [ ]` 开头的行）。

## 2. 逐个处理待办事项

对于每个待办事项：

1. **理解任务**：分析待办事项的内容和要求
2. **执行任务**：
   - 如果是代码改进（如修复 TODO、编写测试），在代码库中进行相应修改
   - 如果是知识记录，准备相应的文档内容
3. **更新勘误文件**：
   - 在 ERRATA.md 的适当位置添加新学到的知识/最佳实践
   - 将已处理的待办事项从 `- [ ]` 改为 `- [x]` 或直接删除
4. **运行测试**：确保修改不破坏现有功能

## 3. 输出格式

处理完成后，汇报：
- 处理了哪些待办事项
- 做了哪些代码修改
- 在 ERRATA.md 中添加了哪些新知识

## 注意事项

- 每处理一个事项就立即更新 ERRATA.md
- 如果某个事项需要用户确认，使用 AskUserQuestion 询问
- 保持 ERRATA.md 的现有格式风格
- 新增的知识应归类到适当的章节（MoonBit 编码规范、WebAssembly 实现等）
