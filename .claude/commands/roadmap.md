# 完成 ROADMAP 任务

完成 ROADMAP.md 中接下来的 $ARGUMENTS 个未完成任务。

## 执行步骤

### 1. 读取 ROADMAP.md
读取 `ROADMAP.md` 文件，找到所有未完成的任务（以 `- [ ]` 开头的行）。

### 2. 选择任务
从第一个未完成的任务开始，选择接下来的 $ARGUMENTS 个任务进行处理。

如果 $ARGUMENTS 为空或未指定，默认处理 **1** 个任务。

### 3. 逐个完成任务

对于每个选中的任务：

1. **分析任务**：理解任务要求，确定需要修改的文件
2. **实现功能**：
   - 在相应文件中添加或修改代码
   - 遵循项目的编码规范（见 ERRATA.md 和 CLAUDE.md）
3. **添加测试**：为新功能编写测试用例
4. **运行测试**：执行 `moon test` 确保所有测试通过
5. **更新 ROADMAP**：将任务从 `- [ ]` 改为 `- [x]`

### 4. 提交更改

完成所有任务后：
1. 运行 `moon fmt && moon info`
2. 使用 git 提交更改，提交信息包含完成的任务列表

### 5. 汇报结果

输出：
- 完成了哪些任务
- 修改了哪些文件
- 新增了多少行代码
- 测试结果

## 注意事项

- 如果某个任务依赖于尚未实现的功能，跳过并说明原因
- 每个任务完成后立即更新 ROADMAP.md
- 保持代码质量，不要为了完成任务而降低标准
- 参考 ERRATA.md 中的最佳实践
- 如果任务描述不清晰，使用 AskUserQuestion 询问

## 示例

```
/roadmap 3
```

这将完成 ROADMAP 中接下来的 3 个未完成任务。
